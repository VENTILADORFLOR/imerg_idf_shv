# GPM IMERG IDF Correction — Sihanoukville

Python scripts for benchmark-referenced bias assessment of GPM IMERG V07
against an official IDF table, with watershed delineation and peak discharge
estimation for a representative drainage node in Sihanoukville, Cambodia.

**Study point:** 10.540139°N, 103.656389°E (Sihanoukville, Cambodia)  
**Satellite data:** NASA GPM IMERG V07 Final Run, half-hourly, 2000–2024  
**Official benchmark:** Sihanoukville IDF table, Department of Hydrology, 1999–2019

## Workflow

### Prerequisites

```bash
pip install -r requirements.txt
earthengine authenticate          # one-time GEE authentication
```

Edit `config/gee_config.py` to set your own GEE project ID if needed.

---

### Step 1 — Extract raw data from Google Earth Engine

```bash
python 01_data_extraction/imerg_2000_2019.py
python 01_data_extraction/imerg_2020_2024.py
python 01_data_extraction/get_alos_dem.py
python 01_data_extraction/get_chirps_daily.py
```

Tasks are submitted to GEE and run in the cloud (approximately 15–25 minutes each).
Download the exported files from Google Drive `GEE_Exports/` into `data/raw/`.

> **Note on IMERG V07 availability:**  
> IMERG V07 Final Run covers through September 2025, then production was suspended
> pending the V08 release (expected summer 2026). The 2020–2024 extraction uses
> `filterDate("2020-01-01", "2025-01-01")` — a right-open interval capturing
> five complete calendar years (2020–2024).

> **Note on CHIRPS collection choice:**  
> Two daily CHIRPS v3 products are available on GEE. `DAILY_SAT` partitions
> pentadal totals using NASA IMERG Late V07 and is therefore unavailable before
> ~1998. `DAILY_RNL` uses ECMWF ERA5 reanalysis and provides the full
> 1981-to-present record. `DAILY_RNL` is used here to obtain the complete
> 1981–2025 time series required for the 44-year Gumbel comparison.

### Step 2 — Compute rolling annual maxima and bootstrap confidence intervals

```bash
python 02_annual_maxima/compute_annual_maxima.py
python 02_annual_maxima/compute_bootstrap_ci.py
```

`compute_annual_maxima.py` outputs written to `data/processed/`:

| File | Description |
|------|-------------|
| `annual_maxima_imerg_2000_2019.csv` | Per-year maxima by duration, 2000–2019 |
| `annual_maxima_imerg_2020_2024.csv` | Per-year maxima by duration, 2020–2024 |
| `summary_2000_2019.csv` | Mean / max / min / std intensity per duration |
| `summary_2020_2024.csv` | Same for 2020–2024 period |
| `comparison_2000_2019_vs_2020_2024.csv` | Side-by-side with change % |
| `IMERG_temporal_stability_comparison.xlsx` | All of the above in one workbook |

`compute_bootstrap_ci.py` outputs written to `data/processed/`:

| File | Description |
|------|-------------|
| `bootstrap_ci.csv` | Bootstrap 95% CI for each duration correction factor (2,000 replicates, L-moment Gumbel, seed=42) |

### Step 3 — Gumbel distribution fitting and 24-hour comparison

```bash
python 03_idf_fitting/gumbel_idf_2000_2019.py
python 03_idf_fitting/compare_24h_gumbel.py
```

`gumbel_idf_2000_2019.py`:  
Input: `data/processed/annual_maxima_imerg_2000_2019.csv`  
Output: `data/processed/imerg_idf_2000_2019_gumbel.csv`

Fits a two-parameter Gumbel distribution by MLE to each duration series and
derives design intensities for 5-, 10-, and 25-year return periods.

`compare_24h_gumbel.py`:  
Inputs: `data/raw/rainfall_point_imerg_2000_2019.csv`, `rainfall_point_imerg_2020_2024.csv`,
`rainfall_point_chirps_1981_2025.csv`  
Outputs: `data/processed/gumbel_24h_comparison.csv`, `data/figures/gumbel_24h_comparison.png`

Compares five 24-hour Gumbel frequency curves: the official benchmark, IMERG
2000–2019, IMERG 2000–2024, CHIRPS 1999–2019, and CHIRPS 1981–2025. The
same-window comparison (first three) isolates satellite bias; the long-record
comparison (last two) tests whether the official 20-year record systematically
overestimates 24-hour design depths.

`diagnose_outliers.py` is a standalone diagnostic script (not required for
the main analysis outputs) that identifies extreme annual maxima events in
IMERG and CHIRPS and cross-checks dates between the two datasets. Outputs
are written to `data/processed/` and `data/processed/outliers_log.txt`.

### Step 4 — Watershed delineation, Kirpich estimate, and catchment figure

```bash
python 04_watershed/delineate_watershed.py
python 04_watershed/compute_kirpich.py
python 04_watershed/plot_catchment_figure.py
```

`delineate_watershed.py` requires:

| File | Path | Availability |
|------|------|-------------|
| ALOS 30 m DEM | `data/raw/sihanoukville_alos30m_dem.tif` | Generate via Step 1 |
| Pour point | `data/drainage_node/drainage_node.shp` | **Not provided** — proprietary site data |

Outputs `data/watershed/catchment.shp`, `data/watershed/flow_paths.shp`, and
`data/watershed/dem_utm48n.tif`.

> **DEM version — AW3D30 V4.1:**  
> The extraction script uses **V4.1** (released April 2024, GEE collection
> `JAXA/ALOS/AW3D30/V4_1`), which corrected anomalous elevation values in
> 19,051 tiles globally. The GEE band name remains **`DSM`** — no script
> changes are required relative to earlier versions.

`compute_kirpich.py` reads the watershed outputs and DEM, and writes
`data/processed/kirpich_tc.csv` containing flow path length, elevation
difference, average slope, and Kirpich (1940) time of concentration.

`plot_catchment_figure.py` additionally requires:

| Layer | Path | Availability |
|-------|------|-------------|
| Site boundary | `data/study_area/study_area.shp` | **Not provided** — proprietary site data |
| Depression polygons | `data/depression/depression.shp` | **Not provided** — proprietary site data |
| Main road | `data/main_road/main_road.shp` | **Not provided** — proprietary site data |
| OSM road network | `data/roads/gis_osm_roads_free_1.shp` | Download from Geofabrik (see below) |

All shapefiles are reprojected to **EPSG:32648** (WGS 1984 UTM Zone 48N) at
load time, regardless of their source CRS.  
Output: `data/figures/catchment_figure.png`

> **OSM road network:**  
> Download the Cambodia extract from [download.geofabrik.de](https://download.geofabrik.de/asia/cambodia.html),
> extract `gis_osm_roads_free_1.shp` and place it in `data/roads/`.

### Step 5 — Generate summary tables

Run Steps 2–4 first to ensure all processed CSVs are current, then:

```bash
python 05_tables/generate_tables.py
```

Inputs required:

| File | Description |
|------|-------------|
| `data/tables/Design_Rainfall_Intensities_Selected.csv` | Selected benchmark intensities — **not included**; create manually (5 durations × 3 return periods, see format in `generate_tables.py`) |

| `peak_discharge.csv` | Rational method peak discharge scenarios |
| `all_tables.xlsx` | All four tables in one workbook |

## Key Findings

| Duration | Correction factor αd | 95% CI |
|----------|----------------------|--------|
| 30 min | 3.65 | 3.02–4.34 |
| 1 h | 2.90 | 2.61–3.32 |
| 2 h | 2.08 | 1.74–2.52 |
| 4 h | 1.69 | 1.36–2.05 |
| 24 h | 1.24 | 0.98–1.47 |

Raw IMERG underestimates short-duration design rainfall by **63–74%** relative
to the official IDF benchmark at 30–60 min durations. After correction, the
10-year 1-hour design intensity increases from 37.8 mm/h to 109.8 mm/h,
raising the estimated peak discharge from 20.8 m³/s to 60.4 m³/s.

The Kirpich time of concentration for the delineated catchment is approximately
**29 minutes** (flow path length 3,646 m, average slope 7.63%). The 1-hour
design duration exceeds this estimate and represents a conservative engineering
choice.

## Reproducibility Note

All IMERG, CHIRPS, and DEM data are static archives. Re-running the Step 1
extraction scripts with the same parameters will produce byte-identical outputs.
The 2020–2024 IMERG series covers five complete calendar years; no partial-year
data is included, ensuring that annual maxima are comparable across years.
Bootstrap confidence intervals are computed with a fixed random seed (`seed=42`)
and are therefore fully reproducible.
